# Revenue Tracker Spec

Purpose: one lightweight source of truth for leads, fit calls, objections, follow-up, payment path, and closed revenue. This is designed to start as a CSV or Google Sheet, then graduate into the admin dashboard once the sales motion is proven.

## Files created in this repo

- `src/_data/revenuePipeline.csv`: import-ready tracker header with no fake contacts.
- `src/_data/dailyMoneyMetrics.json`: daily briefing data contract for visits, opt-ins, calls, bookings, invoices, and sales.
- `src/admin/morning-briefing.njk`: renders the daily money metrics and pipeline tracker status.

## Google Sheet / CSV columns

Required columns:

1. `lead_id`
   - Stable ID generated by the agent or sheet formula.
   - Suggested format: `lead_YYYYMMDD_###`.
2. `created_date`
   - Date the lead entered the tracker.
   - Format: `YYYY-MM-DD`.
3. `source`
   - Where the lead came from.
   - Examples: organic search, newsletter, Cal.com, referral, district list, conference, prior customer, direct outreach.
4. `role`
   - Role or buyer type.
   - Examples: BCBA, district admin, special education director, RBT candidate, parent, clinic owner.
5. `organization`
   - District, agency, company, or school when known.
6. `contact_name`
   - Real person only. Leave blank if unknown.
7. `email`
   - Real email only. Leave blank if unknown.
8. `phone`
   - Real phone only. Leave blank if unknown.
9. `pain`
   - The problem they said or implied.
   - Keep this short and concrete.
10. `offer`
   - Current best-fit offer.
   - Examples: RBT exam prep, BehaviorSchool Pro, Transformation Program, consultation, district training.
11. `interview_status`
   - Allowed values: not_started, requested, scheduled, completed, declined, no_show, not_applicable.
12. `fit_call_status`
   - Allowed values: not_started, invited, booked, completed, no_show, declined, not_qualified.
13. `objection`
   - Main blocker in the buyer's words when known.
14. `follow_up_date`
   - Next promised follow-up date.
   - Format: `YYYY-MM-DD`.
15. `close_result`
   - Allowed values: open, won, lost, paused, nurture, referred, duplicate.
16. `payment_path`
   - How money will move if they buy.
   - Examples: Stripe checkout, invoice, district PO, FreshBooks, App Store, manual ACH.
17. `next_action`
   - One concrete next step.
18. `owner`
   - Person or agent responsible for the next action.
19. `last_touch_date`
   - Last meaningful touch.
   - Format: `YYYY-MM-DD`.
20. `notes`
   - Short notes. Do not put secrets, tokens, or private unrelated details here.

## Daily money metrics contract

The daily briefing should show these metrics every morning:

- visits
- opt-ins
- calls
- bookings
- invoices
- sales

Each metric has:

- `label`: display name.
- `value`: number when verified, `null` when not accessible.
- `source`: system of record.
- `status`: one of `connected`, `manual`, `blocked`, `not_configured`.
- `nextAction`: exact step to make the value real.

Do not fabricate numbers. If a source is not connected, render `Not connected` and show the next action.

## Recommended Google Sheet tabs

1. `Pipeline`
   - Main lead tracker using the CSV header.
2. `Daily Metrics`
   - Date, visits, opt-ins, calls, bookings, invoices, sales, notes.
3. `Sources`
   - Mapping of each metric to its system of record, API/export path, credential status, and owner.
4. `Follow-ups Due`
   - Filtered view where `follow_up_date <= TODAY()` and `close_result = open`.
5. `Won Revenue`
   - Filtered view where `close_result = won`.

## First automation targets

1. Import Cal.com bookings and completed calls once API access is available.
2. Pull Stripe successful payments and open invoices once API access is approved.
3. Pull newsletter/site opt-ins from the current capture source once credentials are available.
4. Pull visits from Search Console, GA, or Netlify Analytics, whichever is already available.
5. Send a morning digest with only verified values and explicit blockers.
